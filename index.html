업무관리 앱(AppSheet) 제작설명서

이 문서 하나만 보고 따라 하면, 앱 개발을 몰라도 업무관리 앱을 직접 만들거나 고칠 수 있습니다.

회사 구글 워크스페이스 + AppSheet(구글 무료 도구)로 만듭니다.

📖 이 문서의 구성 — 두 부분 ① 도입 배경(왜 이 방식을 쓰는지 — 팀과 함께 읽는 부분)과 ② 제작 매뉴얼(실제로 만들고 고치는 법 — 담당자가 보는 부분). 바쁘시면 ①만 읽어도 "이게 왜 필요하고, 우리가 충분히 다룰 수 있는지"를 알 수 있습니다.
📢 PART A · 팀과 함께 읽기
왜 이 방식인가 — 도입 배경
새로운 걸 “배워야 하는” 이야기가 아니라, 이미 쓰는 도구를 더 잘 쓰는 이야기입니다.
30초 요약 — 이것만 읽어도 됩니다
무엇을
회사가 이미 쓰는 구글 도구 위에, 업무 상태(누가 어디까지)·업무별 의견(댓글)·변경 기록을 다루는 화면을 하나 더합니다.
왜
예전 Flow에 있던 “업무관리”가 구글로 옮기며 빠졌습니다. 채팅은 대화가 묻히고, 개인 메모는 팀과 공유되지 않습니다.
어디서
폰(안드로이드·아이폰)과 컴퓨터(윈도우·맥) 어디서나 씁니다. 같은 회사 계정으로 들어가면 같은 데이터입니다.
내 부담
새 프로그램을 배우거나 새로 가입할 것은 없습니다. 데이터는 우리 구글 표에 그대로 남고, 마음에 안 들면 화면(앱)만 꺼도 됩니다.
비용
회사가 이미 내는 요금에 포함된 기능이라, 이 용도로 추가 요금은 들지 않습니다.
아래는 “왜 이렇게 하는지”와 “어떻게 생겼는지”가 궁금한 분을 위한 설명입니다. 담당자용 제작 방법은 문서 뒷부분(제작 매뉴얼)에 있습니다.

1. 우리가 겪은 불편 — 무엇이 빠졌나

Flow에서 구글 워크스페이스로 옮기며 메일·문서공유·드라이브·일정·AI(제미나이·노트북LM)는 분명히 좋아졌습니다. 그런데 한 가지가 사라졌습니다 — “업무관리”입니다.

누가 무슨 일을 어디까지 했는지(상태), 그 일에 대한 의견·피드백(업무별 댓글), 누가 언제 무엇을 바꿨는지(추적) — 이게 안 보입니다. 구글 Tasks는 개인 메모, 채팅은 대화가 위로 흘러가 묻혀버립니다.

핵심 구글 워크스페이스는 일부러 “업무관리(PM)” 역할을 하지 않습니다. 고장이 아니라 설계 의도입니다 — 워크스페이스는 소통·문서·저장 도구이지, 업무를 카드로 관리·추적하는 도구가 아닙니다.
워크스페이스가 잘하는 것빠진 것 (= Flow가 해주던 업무관리·PM)
메일 · 문서 공동작업 · 드라이브 · 캘린더 · 화상회의 · AI업무 상태 공유(진행·완료) · 업무별 의견(댓글) · 담당·추적

그래서 전 세계 회사들은 워크스페이스 “위에” 업무관리 도구를 하나 얹어서 씁니다. 우리도 그 한 가지만 채우면 됩니다.

2. 왜 AppSheet + Apps Script인가 — 당위성

외부의 또 다른 유료 서비스를 새로 들이는 대신, 이미 우리가 매달 돈 내고 쓰는 구글 워크스페이스에 포함된 두 도구로 해결합니다.

도구역할우리와의 관계
AppSheet구글 시트를 앱 화면·알림으로 (코딩 없이 설정만으로)구글이 만든 도구. 우리 워크스페이스(유료)에 포함 → 추가 비용 없음
Apps Script자동화 (알림 발송·반복 작업 등)우리 팀이 이미 구글 시트에 쓰고 있는 바로 그 기술
한 줄 요약 추가 지출 0원. 이미 내는 요금 안에 들어있는 숨은 기능을 켜서 100% 활용하는 것 — 우리 작업 환경에 가장 빠르고 효율적으로 녹아드는 방법입니다.

덧붙여, 이건 우리만의 실험이 아닙니다. 구글 워크스페이스를 쓰는 수많은 회사가 정확히 이 조합(워크스페이스 + AppSheet)으로 업무관리를 운영하고 있습니다 — 이미 검증된 길입니다.

정확히 알아둘 점 현장 사진·바코드·오프라인·앱 푸시 알림은 폰의 설치 앱에서 됩니다. 컴퓨터 브라우저에서는 이 기능들이 안 됩니다. 그래서 알림은 이메일을 기본으로 보냅니다(폰·컴퓨터 어디서나 수신). “인터넷이 없어도 된다”는 아닙니다.

참고: 다른 회사들은 무엇을 쓰나 — 그리고 비용

“워크스페이스 위에 얹는 업무관리 도구”는 크게 글로벌 SaaS와 국내 협업툴이 있습니다. 모두 좋은 도구지만, 공통적으로 인당 월 구독료가 들고 · 데이터가 외부 서비스에 저장되며 · 또 다른 로그인·벤더가 늘어납니다.

우리의 판단 위 도구들은 검증된 좋은 제품이고, 회사가 더 커지거나 고급 기능이 꼭 필요해지면 충분한 선택지입니다. 다만 우리 상황(이미 유료 워크스페이스 사용 + 이미 Apps Script 운용)에서는 — 같은 목적(업무 상태·댓글·추적)을 추가 비용 0원 · 데이터는 우리 시트 · 새 벤더/로그인 없이 달성하는 AppSheet 방식이 가장 합리적입니다. 위 도구들은 연 수십~수백만원의 구독료와 데이터 외부 저장이 따라옵니다.
정리 선택지는 열려 있되, 우리에겐 이미 값을 치른 도구(워크스페이스)를 활용하는 것이 가장 빠르고 저렴합니다. 유료 도구는 “정말 필요해지면” 그때 검토해도 늦지 않습니다.

3. “이거, 우리가 계속 다룰 수 있나?” — 통제권

가장 중요한 질문입니다. 답은 “네, 우리 손 안에 있습니다.” 이유는 4가지입니다.

  1. 데이터는 늘 쓰던 구글 시트 그대로. 직원·업무 정보를 구글 스프레드시트에서 직접, 엑셀처럼 대량으로 작성·수정·붙여넣기 할 수 있습니다. 앱은 그 시트를 보기 좋게 띄우고 알림을 더하는 “창”일 뿐, 데이터를 따로 가두지 않습니다.
  2. 이미 다루는 기술의 연장. 우리 팀은 이미 Apps Script를 구글 시트에 적용하고 있습니다. 완전히 새로운 미지의 기술이 아니라, 지금 쓰는 도구를 한 걸음 넓히는 것입니다.
  3. 코딩 없이 “설정”으로. AppSheet 화면 만들기는 메뉴에서 고르고 채우는 작업입니다. 프로그래밍이 아닙니다(이 매뉴얼의 Part B가 그 증거 — 따라 하면 됩니다).
  4. 되돌릴 수 있고, 특정인에 묶이지 않습니다. 마음에 안 들면 앱을 꺼도 데이터(시트)는 그대로 안전합니다(락인 없음). 데이터가 표준 구글 시트라 담당자가 바뀌어도 누구나 운영 가능 — 그래서 이 매뉴얼을 남깁니다.
결론 새로운 시스템을 “배워야 하는” 부담이 아니라, 이미 회사가 유료로 쓰는 도구를 더 잘 쓰는 일입니다. 데이터·통제권은 처음부터 끝까지 우리 손에 있습니다.

앱이 어떻게 만들어지는지 큰 그림입니다. (외울 필요 없음, 이해만)

용어쉽게 말하면
구글 시트 (데이터 저장소)업무·직원·댓글이 표(엑셀)로 저장되는 곳. 진짜 데이터는 여기 다 있습니다.
AppSheet (앱)그 구글 시트를 읽어서 휴대폰/PC 앱 화면으로 보여주는 도구. 시트가 바뀌면 앱도 바뀝니다.
테이블 / 컬럼 / 뷰테이블=시트의 한 탭, 컬럼=표의 열, 뷰=앱에서 보이는 화면(목록·상세·폼 등).
핵심 데이터는 구글 시트에, 화면은 AppSheet에. 둘은 항상 연결돼 있습니다. 그래서 시트를 고치면 데이터가, AppSheet을 고치면 화면이 바뀝니다.

사전 준비

  1. 회사 구글 계정(@example.com)으로 로그인돼 있어야 합니다. (개인 gmail 아님 — 알림·사진 기능이 회사 계정에서만 제대로 작동)
  2. 구글 워크스페이스 유료 플랜이면 AppSheet이 무료로 포함돼 있습니다. (관리자에게 "AppSheet Core 라이선스 있는지" 확인 가능)
  3. 크롬(Chrome) 브라우저 권장.
⚠️ 가장 중요한 주의 모든 작업은 회사 계정으로 하세요. 화면 오른쪽 위 동그란 프로필이 회사 계정(예: 홍길동 / user@example.com)인지 항상 확인.
PART 1 완료

백엔드 구글 시트 만들기

업무·직원·댓글을 담을 구글 시트를 먼저 만듭니다. 탭 3개, 각 탭의 첫 줄에 정해진 제목(헤더)을 넣습니다.

1-1. 새 시트 만들고 이름 정하기

  1. 회사 계정으로 sheets.google.com 접속 → + 빈 스프레드시트
  2. 왼쪽 위 제목 없는 스프레드시트를 클릭하고 HJ_업무관리 로 이름 변경

1-2. 탭 3개 만들기

시트 아래쪽 탭 이름을 바꾸고, + 버튼으로 탭을 추가합니다. (탭 이름을 더블클릭하면 수정됨)

HJ_업무관리 — Google Sheets
+ G_직원 ▾ F_Tasks ▾ F_Comments ▾
↑ 시트 맨 아래 탭 영역. 이름을 정확히 G_직원 / F_Tasks / F_Comments 로.

1-3. 각 탭 1행에 헤더(열 제목) 입력

각 탭을 열고 A1 셀부터 오른쪽으로 아래 제목들을 한 칸씩 입력합니다. (한 칸 입력 후 Tab 키로 옆 칸 이동)

탭1행에 넣을 제목(왼쪽부터 순서대로)
G_직원직원_ID · 이름 · 이메일 · 부서 · 직책 · 휴대전화 · 재직상태

1-4. G_직원에 직원 명부 넣기

  1. G_직원 탭 2행부터 직원을 한 명씩 입력. 직원_ID는 E0001, E0002… 처럼 겹치지 않게.

예시 데이터 E0001 · 홍길동 · user@example.com · 관리 · 과장 · (빈칸) · 재직

PART 4 진행 중

컬럼 타입 설정

AppSheet이 자동으로 정한 컬럼 종류(타입)를 우리 설계에 맞게 바로잡습니다. 이게 앱이 똑똑하게 작동하는 핵심입니다.

4-0. 컬럼 편집하는 기본 방법 (공통)

  1. 왼쪽 Data 목록에서 테이블 선택 (예: F_Tasks)
  2. 가운데 컬럼 목록에서 바꿀 컬럼 왼쪽의 연필(✏️) 아이콘 클릭 → 설정 창이 열림
  3. Type 칸을 클릭하고 타입 이름을 그대로 타이핑하면 자동 선택됩니다 (예: Ref, Enum, Image)
  4. 아래 Type Details에서 세부 설정 → 오른쪽 위 Done
  5. 모든 변경 후 오른쪽 위 파란 SAVE로 저장
💡 꿀팁 Type 칸은 드롭다운이 잘 안 보일 때가 있는데, 클릭 후 그냥 타입 이름을 입력하면 됩니다(예: "Ref" 입력 → Ref 선택됨). Source table 등 다른 선택 칸도 같은 방식.

PART 5 완료

화면(뷰) 만들기 · 다듬기

데이터(테이블·컬럼)가 준비되면, 사람이 보는 화면(뷰)을 만듭니다. AppSheet 왼쪽 메뉴의 📱 UX(휴대폰 아이콘)에서 작업합니다. 아래 순서대로 하면 "내 업무 / 업무 보드 / 댓글"이 보기 좋게 정리됩니다.

5-0. 화면 편집 기본 (공통)

  1. 왼쪽 세로 메뉴에서 📱 (UX) 아이콘 클릭 → 화면(View) 목록이 나옵니다
  2. PRIMARY NAVIGATION = 앱 아래쪽 탭에 보이는 주요 화면들. 여기 화면을 고르면 오른쪽에서 설정합니다
  3. 새 화면은 PRIMARY NAVIGATION 옆 + → Create a new view
  4. 각 화면에서 정하는 것: View name(이름) · For this data(어떤 테이블/슬라이스) · View type(deck/table/detail/form 등)
  5. 바꾼 뒤 오른쪽 위 파란 SAVE
💡 뷰 타입 한눈에 deck=카드 목록(추천) · table=표 · detail=한 건 상세 · form=입력 폼 · dashboard=여러 화면 한 번에(보드용).

5-1. 표시값(라벨) — 번호 대신 이름·제목이 보이게 완료

설정 안 하면 담당자가 E0002, 업무가 965cd804처럼 식별번호로만 보입니다. 각 테이블에 "이 줄을 대표하는 컬럼(라벨)"을 정해주면 이름·제목으로 바뀝니다.

  1. Data → 테이블 → 컬럼 연필 → 아래로 스크롤 → Display 섹션 → Label 체크
  2. G_직원 라벨 = 이름, F_Tasks 라벨 = 제목, F_Comments 라벨 = 댓글표시(아래 5-4)로 지정
⚠️ 라벨은 테이블당 1개 새 라벨을 체크하기 전에 기존 라벨을 먼저 해제하세요. 안 그러면 AppSheet이 새로 체크한 걸 자동으로 풀어버립니다. (순서: 기존 라벨 해제 → 원하는 컬럼 Label 체크 → SAVE)

5-2. "내 업무" 화면 — 본인 담당 업무만 완료

현장 직원이 앱을 열면 자기에게 배정된 업무만 보이게 하는 화면. 먼저 "필터(슬라이스)"를 만들고, 그 위에 화면을 올립니다.

  1. UX → PRIMARY NAVIGATION + → Create a new view → 이름 내 업무
  2. For this data를 F_Tasks로 바꾼 뒤 → Use slices to filter your data → Create a new slice for F_Tasks
  3. 슬라이스 이름 내업무, Row filter condition 칸에 식 입력:
    [담당자].[이메일] = USEREMAIL() (담당자의 이메일이 = 지금 로그인한 사람) → Done
  4. 다시 "내 업무" 화면에서 For this data = 내업무(슬라이스) 확인, View type = deck → SAVE
💡 식 입력 꿀팁(중요) 식은 인라인 "=" 칸에 직접 타이핑하면 들어갑니다. 큰 "Expression Assistant" 창의 넓은 입력칸은 붙여넣기가 잘 안 될 수 있으니, 작은 "=" 칸에서 입력하세요. 한글 컬럼명 [담당자]도 그대로 타이핑됩니다.
슬라이스 — 내업무
Source Table
F_Tasks
Row filter condition
[담당자].[이메일] = USEREMAIL()
이렇게 하면 "내 업무" 화면에 로그인한 본인 담당 업무만 나옵니다.

5-3. "업무 보드" — 상태별로 묶어 보기 완료

업무 목록을 요청 / 진행중 / 완료처럼 상태별로 묶어 보드처럼 보이게 합니다. (AppSheet엔 드래그 칸반이 없으니, "상태별 그룹화"로 같은 효과를 냅니다.)

  1. UX → F_Tasks 화면 선택 → 아래로 스크롤 → View Options
  2. Group by → Add → 컬럼을 상태로 선택 → SAVE
결과 업무들이 상태 머리글 아래로 묶여 표시됩니다. (정렬을 요청→진행중→완료 순서로 더 다듬을 수 있음 — 다음 작업.)

5-4. 댓글 화면 정리 — 카드로, "작성자 · 내용" 한 줄로 완료

업무 상세 맨 아래 "댓글" 영역을 보기 좋게. ① 표→카드, ② 영역 이름을 "💬 댓글"로, ③ 카드에 작성자 이름이 보이게.

① 카드(deck)로 바꾸기 + 이름

  1. UX → SYSTEM GENERATED → F_Comments_Inline 화면 선택 → View type = deck → SAVE
  2. 영역 이름 바꾸기: Data → F_Tasks → 맨 아래 Related F_Comments 컬럼 연필 → Display → Display name 칸에 "💬 댓글" 입력 → Done → SAVE

② 카드에 작성자 표시 — 가상컬럼 만들기

  1. Data → F_Comments 테이블 → 오른쪽 위 ⋮ 메뉴 → Add virtual column
  2. 새로 생긴 New Virtual Column 연필 → 이름 댓글표시
  3. App formula 칸(인라인 "=")에 입력: CONCATENATE([작성자].[이름], " · ", [내용]) → 저장
    ⚠️ 반드시 [작성자].[이름](점 이름)로! 그냥 [작성자]만 쓰면 이름이 아니라 직원번호(E0001)가 나옵니다 — 참조 컬럼은 점(.)으로 표시할 컬럼을 지정해야 함.
  4. F_Comments 라벨을 내용 → 댓글표시로 교체(5-1 방식: 내용 라벨 해제 → 댓글표시 Label 체크) → SAVE

댓글이 "작성자 · 내용" 카드로 깔끔하게. Add로 누구나 댓글 추가.
🔴 가장 중요한 함정 (TRAP-004) — 댓글 키 댓글(F_Comments)의 키(Key)는 반드시 Comment_ID여야 합니다. 만약 키가 부모인 Task_ID로 잡히면(자동추론이 그렇게 하기 쉬움), 한 업무에 댓글이 1개밖에 안 달립니다(두 번째부터 "There is already a row with the key" 오류).
조치: Data → F_Comments → Comment_ID = Type Text + Key 체크 + Initial value UNIQUEID() + (폼에서 숨기려면 Show? 해제), 그리고 Task_ID의 Key는 해제(참조는 유지). KEY 설정은 컬럼 목록을 오른쪽으로 스크롤하면 나오는 KEY? 체크박스에서.
